# Руководство по переносу макета в код

## Как использовать DesignConstants.swift

Файл `DesignConstants.swift` содержит все параметры дизайна экрана. Чтобы перенести значения из макета (Figma, Sketch, Adobe XD и т.д.), просто замените значения в этом файле.

## Пошаговая инструкция

### 1. Откройте макет в вашем дизайн-инструменте
- Figma
- Sketch
- Adobe XD
- или другой инструмент

### 2. Измерьте значения в макете

#### Отступы (Spacing)
- **horizontalPadding**: Измерьте отступы слева и справа от края экрана до контента
- **titleBottom**: Расстояние от заголовка до следующего элемента
- **infoBottom**: Расстояние от информационного текста до примеров
- **templatesBottom**: Расстояние от примеров до поля ввода
- **buttonBottom**: Расстояние от кнопки до нижнего края

#### Типографика (Typography)
- **titleSize**: Размер шрифта заголовка (в пунктах)
- **titleWeight**: Толщина шрифта заголовка (light, regular, medium, bold)
- **infoSize**: Размер шрифта информационного текста
- **templateSize**: Размер шрифта примеров вопросов
- **inputSize**: Размер шрифта в поле ввода
- **buttonSize**: Размер шрифта кнопки

#### Цвета (Colors)
- **background**: Цвет фона (обычно белый или #FFFFFF)
- **title**: Цвет заголовка
- **info**: Цвет информационного текста
- **template**: Цвет примеров вопросов
- **input**: Цвет текста в поле ввода
- **button**: Цвет текста кнопки

#### Размеры (Sizes)
- **inputHeight**: Высота поля ввода
- **cursorWidth**: Ширина курсора
- **cursorHeight**: Высота курсора
- **buttonBorderWidth**: Толщина рамки кнопки

### 3. Замените значения в DesignConstants.swift

Откройте файл `DesignConstants.swift` и замените значения:

```swift
// Было:
static let horizontalPadding: CGFloat = 34

// Стало (если в макете 28px):
static let horizontalPadding: CGFloat = 28
```

### 4. Для цветов из макета

Если в макете указан цвет в формате HEX (например, #FF5733):

```swift
// В DesignConstants.swift:
static let info: Color = Color(hex: "FF5733")
```

Или используйте RGB:

```swift
static let info: Color = Color(red: 1.0, green: 0.34, blue: 0.20)
```

### 5. Проверьте результат

После замены всех значений:
1. Запустите приложение
2. Проверьте, что все элементы соответствуют макету
3. При необходимости скорректируйте значения

## Примеры измерений

### В Figma:
1. Выберите элемент
2. Посмотрите значения в панели справа (Properties)
3. Скопируйте значения (например, Font size: 18, Padding: 34)

### В Sketch:
1. Выберите элемент
2. Посмотрите значения в Inspector справа
3. Скопируйте значения

### В Adobe XD:
1. Выберите элемент
2. Посмотрите значения в панели Properties
3. Скопируйте значения

## Важные замечания

- **Единицы измерения**: В макетах обычно используются пиксели (px), в SwiftUI - пункты (pt). Для iOS они обычно совпадают (1px = 1pt для @1x)
- **Разрешение**: Убедитесь, что макет создан для правильного разрешения iPhone (например, 375x812 для iPhone X/11/12 mini)
- **Safe Area**: Не забудьте учесть safe area (вырез экрана, индикатор дома)

## Быстрая проверка

После переноса значений проверьте:
- ✅ Все тексты влезают на экран
- ✅ Отступы соответствуют макету
- ✅ Размеры шрифтов правильные
- ✅ Цвета совпадают
- ✅ Элементы не перекрываются

## Настройка кастомных шрифтов

Если вы используете кастомные шрифты:

1. **Добавьте файлы шрифтов** в папку `Fonts/` проекта
2. **Настройте Info.plist**:
   - В Xcode выберите проект в навигаторе (синяя иконка вверху)
   - Выберите Target "i-ching"
   - Перейдите на вкладку **Info**
   - В разделе **Custom iOS Target Properties** нажмите **+**
   - Добавьте ключ: `Fonts provided by application` (или `UIAppFonts`)
   - Добавьте элемент: `7fonts.ru_ChinaCyr.ttf`

3. **Используйте в коде**:
```swift
Font.custom("ChinaCyr", size: 24)
// или полное имя, которое нужно определить из файла шрифта
```

**Примечание**: Имя шрифта в коде может отличаться от имени файла. Чтобы узнать точное имя:
- Откройте файл .ttf в Font Book (macOS)
- Имя шрифта будет указано вверху
